# Day2Sleep MVP — AI IMU Classification Version

해커톤 시연용 모바일 웹앱 프로토타입입니다. 이번 버전은 기존 임계값 기반 정지/활동 감지에 더해, 사용자가 직접 수집한 IMU 샘플로 앉기·서기·눕기·걷기를 분류하는 브라우저 내 AI 기능을 포함합니다.

## 포함 기능

1. 스마트폰 IMU 기반 실시간 센서 수집
2. AI 학습 탭에서 앉기/서기/눕기/걷기 샘플 수집
3. 브라우저 내 k-NN 분류기로 현재 자세/활동 예측
4. AI 예측 결과 기반 sedentary bout 계산
5. 통증/피로 설문 기반 Body Load Score 산출
6. Body Load Score 기준 교정 루틴 추천
7. IMU 기반 루틴 수행 여부 확인
8. 수면 질 간이 입력과 낮 좌식 패턴 피드백 화면

## AI 분류 방식

- 서버나 별도 API 없이 브라우저에서 작동합니다.
- `DeviceMotionEvent`로 가속도계와 자이로 데이터를 수집합니다.
- 최근 약 2초 구간의 IMU 데이터에서 다음 특징값을 추출합니다.
  - 가속도 변화량 평균/표준편차/최댓값
  - 자이로 회전량 평균/표준편차
  - x/y/z축 중력 방향 비율
  - pitch, roll 기울기
  - jerk 평균
- 사용자가 라벨링한 샘플과 현재 특징값 사이의 거리를 계산합니다.
- 가장 가까운 샘플들을 k-NN 방식으로 투표해 앉기, 서기, 눕기, 걷기를 예측합니다.

## 실행 방법

### PC에서 보기

`index.html` 파일을 브라우저로 열면 됩니다. PC에는 IMU 센서가 없으므로 AI 탭에서 `PC 시연용 예시 AI 데이터 넣기`를 누른 뒤 홈의 `앉아있기 데모`, `걷기 데모` 버튼으로 시연합니다.

### 스마트폰에서 센서 시연

1. 파일을 GitHub Pages, Netlify, Vercel 등 HTTPS 환경에 올립니다.
2. 스마트폰 브라우저로 접속합니다.
3. `IMU 측정 시작` 버튼을 누르고 센서 권한을 허용합니다.
4. AI 탭에서 앉기, 서기, 눕기, 걷기를 각각 선택하고 `6초 샘플 수집`을 진행합니다.
5. 학습 샘플이 쌓이면 홈 화면의 현재 상태가 AI 예측 결과로 표시됩니다.

## 발표 시연 순서

1. AI 탭에서 `PC 시연용 예시 AI 데이터 넣기` 또는 스마트폰 실제 샘플 수집
2. 홈 화면에서 `앉아있기 데모`를 눌러 AI 예측과 sedentary bout 확인
3. `걷기 데모`를 눌러 좌식 중단 확인
4. 점수 탭에서 통증/피로 설문 입력 후 Body Load Score 계산
5. 루틴 탭에서 교정 루틴 추천 확인
6. `수행 확인 시작` 후 걷기 데모를 눌러 수행 완료 확인
7. 수면 탭에서 수면 만족도와 수면 시간 입력 후 피드백 확인

## 발표 때 설명하면 좋은 문장

“이번 MVP에서는 고정 임계값만 사용하지 않고, 사용자가 직접 라벨링한 IMU 샘플을 기반으로 앉기·서기·눕기·걷기를 분류하는 경량 AI 모델을 브라우저 내부에 구현했습니다. 최근 2초간의 가속도·자이로 특징값을 추출하고, k-NN 방식으로 현재 상태와 가장 유사한 학습 샘플을 찾아 자세를 예측합니다.”

## 주의

- 이 MVP는 의료 진단용이 아니라 웰니스 해커톤 시연용입니다.
- 스마트폰 위치가 바뀌면 IMU 패턴도 바뀌므로, 실제 시연 전 같은 위치에서 학습 샘플을 다시 수집하는 것이 좋습니다.
- 현재 sedentary bout 기준은 데모 편의를 위해 20초로 설정되어 있습니다. 실제 서비스에서는 30분 또는 60분 기준으로 조정할 수 있습니다.
